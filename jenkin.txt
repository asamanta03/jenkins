Q1: What is Continuous Integration (CI)?
A: Continuous Integration is a process where developers push code to a centralized location (like GitHub), and automatically building, compilation, integration, and testing happens. The code is tested using unit test cases, and feedback is automatically sent to the developer. This entire automated process is known as CI.

Q2: What is the difference between Agile and CI?
A: In Agile, coding, building, and integrations happen, but automated testing does not occur. In CI, automatic testing happens with test cases and performance testing. When testing is automated, we call it Continuous Integration.

Q3: What is the difference between traditional approach and CI approach?
A: In traditional approach, developers have to wait for managers or leads to manually perform building, compilation, and testing activities, which may happen only 1-2 times per day. In CI approach, tools like Jenkins automatically perform these activities multiple times a day as soon as developers push code, eliminating waiting time.

Q4: Why is Jenkins popular?
A: Jenkins is popular because: 1) It is open source and freely available, 2) It has 2000+ community-developed plugins supporting integrations with various tools, 3) It is fast and reliable, 4) It works on every operating system with the same UI, 5) Approximately 70-80% of companies use Jenkins for CI/CD.

Q5: What is the default port number for Jenkins?
A: The default port number for Jenkins is 8080.

Q6: How does Jenkins integrate with other tools?
A: Jenkins integrates with other tools through plugins. There are 2000+ community-developed plugins available that allow Jenkins to integrate with various tools like GitHub, Kubernetes, SonarQube, Docker, Ansible, etc.

Q7: What are the benefits of using CI?
A: Benefits include: 1) Improved quality through multiple automated testing, 2) Increased productivity as no waiting time is required, 3) Reduced risk because automatic testing happens multiple times, 4) Faster feedback to developers.

Q: What is Java?
A: Java is a high-level, object-oriented, class-based programming language. It is platform-independent, meaning code written once can run on any machine (Windows, Linux, Mac). Q: What is Spring Boot?
A: Spring Boot is a Java framework used to build standalone web-based applications with minimal code. It has auto-configuration and embedded web servers (like Tomcat), so developers don't need to integrate separate web servers. It is an extension of the Spring Framework and provides production-ready code with ready-made frameworks.

Q: What is Maven?
A: Maven is a build automation and dependency management tool primarily for Java-based applications. It automatically downloads required dependencies and manages versions through the POM.xml (Project Object Model) file. Without Maven, developers would need to manually download dependencies, which could lead to version mismatches.

Q: What is the difference between Spring and Spring Boot?

Q: What is POM.xml?

Q: What is unit testing?

Q: What command is used to build with Maven?
A: The command is "mvn package". Using "mvn clean package" will first clean previous dependencies and then create a fresh build.
